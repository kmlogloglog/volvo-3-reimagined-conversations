How it works

One container, two frontends, one shared database.

The big picture

Vän has two screens that work together. A dashboard where the dealer team sees customer profiles, analytics, and campaign data — and a conversational AI (Freja) that customers interact with through voice and text. Both live inside the same application and share the same customer database.

■

Dashboard

The admin screen. Dealer staff see customer profiles, affinities, propensity scores, and live conversations. It reads from the shared database in real time.

■

Conversational AI (Freja)

The customer-facing voice and chat assistant. Powered by Google Gemini, it answers questions about Volvo cars, books test drives, and finds nearby dealers. It writes back to the same shared database.

■

Shared Database

One Google Cloud Firestore database holds everything — customer profiles, conversation history, and agent memory. Both screens connect to the same data, so the dashboard always reflects what Freja is doing.

■

Preview Mode

The split-screen layout shows both the dashboard and conversational AI side-by-side. This is the primary demo view — you're one click away from it via the "Back to Preview" link above.

Why this matters: The dealer team can see exactly what Freja is discussing with customers in real time. When a customer asks about the EX90, the dashboard can immediately surface that interest — no manual data entry, no delays.

2 — Shared Firestore Database

Key insight: Both the React dashboard and the Python agent connect to the same Firestore database (volvo-vaen-v2-db) in GCP project vml-map-xd-volvo. The dashboard reads profiles and live state via the Firebase JS SDK (client-side). The agent writes session history and memories via the Admin SDK (server-side). This means the dashboard reflects real-time agent activity without any extra plumbing.

3 — Request Routing

How preview.html works: It's a simple HTML page with two iframes — the left loads /dashboard/ (React), the right loads /index.html (Nuxt). Both run on the same origin inside Cloud Run. The Nuxt UI opens a WebSocket to /ws/ for real-time agent communication.
